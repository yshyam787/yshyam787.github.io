My First XWiki Development and Integration

Since XWiki is an open-source platform and offers powerful and flexible solutions for knowledge base management, I have recently implemented it in one of my projects. Information on the knowledge base was scattered throughout the website, and there was no centralized solution for this scenario. Thus, XWiki seemed to be an ideal solution. However, I also faced many challenges during the whole process. In this guide, I will walk you through all the steps, and I hope it will help you if you encounter similar issues.

1. Installing XWiki

I installed XWiki using Docker on an Ubuntu Virtual Machine. The Docker Compose file was used to simplify the deployment. All the steps involved in this process, along with the contents of the Docker Compose file, are given here. Please read it carefully and go through all the steps. My XWiki instance was set up with Tomcat and PostgreSQL. However, you can use the services according to your needs.

2. SSL Certificate Generation

This is an optional step. Since my VM had no SSL certificates, my XWiki instance was flagged with a Not Secure warning. Therefore, I installed SSL certificates on Docker from Let's Encrypt using Caddy. The domain name derived from the IP of the VM was used for it.

3. XWiki Instance and REST API

After running the XWiki and PostgreSQL containers and installing the Standard Flavor of XWiki, numerous knowledge bases with sub-branches were created. ⚠ However, I struggled a lot with XWiki API detection. Make sure that you are in the xwiki namespace. Otherwise, you need to use the URL localhost:<your_port_number>/rest/. Many people get confused with localhost:<your_port_number>/xwiki/rest/. The XWiki documentation is not clear enough on this matter.

4. Custom WordPress Plugin Development for XWiki Integration

No suitable and free plugins were available to integrate the XWiki API with WordPress. Thus, I created my own PHP plugin, function.php, and was able to integrate it into WordPress. Before integration, I initially installed Common Markup 1.2 on XWiki for rendering it in HTML. Thus, the content was successfully fetched by WordPress.

5. Parsing Fetched Content

Finally, the fetched content was parsed using Parsedown.php, which was downloaded and added to function.php. (Read more). After parsing, the displayed content contained strings of PHP code. These strings were stripped by creating a function in the PHP plugin. Finally, the displayed content was formatted similarly to how it was written in XWiki.

Note: If you want to transfer data from any website, format, or WordPress to XWiki, Pandoc seems to be the best solution for it.
